# AR花火ビューア｜東京の花火大会

スマートフォンのブラウザで、**東京の主要な花火大会＋ディズニーの花火の実データ**をもとに、
**端末カメラの映像に花火の3DアニメーションをAR合成**します。端末のGPSで現在地を取得し、打上場所
までの **方位・距離** を計算。コンパス（方位センサー）で **実際の方角** に花火を配置し、距離に
応じた **見かけの大きさ** で描画して「いま自分がいる場所からの見え方」を忠実に再現します。

## 方向ガイド
花火がどの方角かを画面内の立て札（常にこちらを向く）で案内します:
- **花火の方角**: 大きな **↓**＋その上に**大会名・距離**
- **90°右**: **←**（花火はあなたの左）／ **90°左**: **→**（花火はあなたの右）
- **真後ろ**: 「**後ろ**」（花火は後ろ）

各大会で打上場所・号数構成（=高さ/大きさ）が変わるため、現在地からの見え方も変化します。

### 地域の切替（全国対応の下地）
データは **地域(`region`)** を持ち、リスト/マップの「地域」セレクトで絞り込みます。現状は
**関東のみ**。`js/festival-data.js` の `REGIONS` に地域を足し、その地域の大会を追加すれば、
UIの選択肢は自動で増えます（全国・他地方への拡張を想定した構造）。

> 開催日・会場は各大会公式／自治体／Wikipedia 等の一次情報に基づきます（年により変動。
> 直近開催回を採用し、過去実績は「※年」を併記）。座標は会場の代表点で数百m程度の誤差を含みます。

## 画面構成（リスト／マップ／AR）
- **リスト画面**: 大会を **日付順／距離順** で並べ替え。各カードの［ARで見る］でAR起動。
  ［🗺 マップを見る］でマップ画面へ。
- **マップ画面**: 地図（OpenStreetMap）上に大会の打上場所をピンで表示。**ピンをタップ**すると
  大会名・日付のポップアップが出て、［ARで見る］でAR起動。［← リストへ戻る］でリストへ。
- **AR画面**: カメラ映像に花火を合成。［≡ 一覧］でリストに戻れる。

## 使い方（スマホ）
1. GitHub Pages で公開されたHTTPSのURLをスマホのブラウザで開く
2. リストで **日付順／距離順** を選び大会を探す（または［マップを見る］→ ピンをタップ）
3. ［ARで見る］→ **日本語の許可ダイアログ** で［許可する］→ 続く端末の確認（カメラ／モーションと画面の向き／位置情報）も許可
4. **スマホを傾けて**（操作はジャイロのみ・スワイプ不要）、↓/←/→/後ろ のガイドに従って花火の方角へ向ける
5. 画面下の **🧭 方位表示**（今向いている方角／花火の方角）も目安に

> センサー・カメラは **HTTPS** 上でのみ動作します（ローカルは `http://localhost` 可）。
> アプリ内ブラウザ（SNS等）はカメラが使えないことがあります → Safari / Chrome で開いてください。

## 仕組み（忠実再現）
- **AR合成**: `getUserMedia` の背面カメラ映像を全画面背景に敷き、A-Frame の透過キャンバス
  （`renderer="alpha: true"`）に花火を加算合成。
- **アスペクト比ロック（`aspect-lock`）**: 3D透視投影カメラのアスペクト比を毎フレーム実画面
  （`window.innerWidth/Height`）へ強制同期し、花火（球＝円）が横長に潰れるのを防ぐ。
- **ジャイロ見回し（`gyro-look`）**: 端末の傾き(`deviceorientation`)からカメラ姿勢を決定（スワイプ
  操作なし）。A-Frame標準の look-controls を使わないため、英語の許可ダイアログも出さない。
- **実スケール座標系**（1 unit = 1 m）。視点をワールド原点に置き、各会場を
  方位・距離から `x = d·sinθ, z = -d·cosθ`（北=-Z）に配置、開花を実際の高度 `y = h` に置く。
  開花球の半径＝開花直径/2[m]。遠近投影により **見かけの画角（大きさ）が自動的に正しく** なる。
- **真北整合**: iOS `webkitCompassHeading` / Android `deviceorientationabsolute` から方位を取得し、
  花火ワールドを一定角回転させて北を実際の北へ合わせる（一度だけ確定）。画面下に現在の方位を表示。
- **許可ダイアログ**: 日本語の説明＋ボタンの自前モーダルで意図を提示してから、端末の許可を要求。
- **方向ガイド**: 花火の方角・±90°・真後ろに、常にこちらを向く立て札（日本語はcanvasテクスチャ）を
  配置し、↓/←/→/後ろ で花火の方向を案内。
- **音の遅延**: 開花の光から `距離 / 音速(340m/s)` だけ遅れて「ドン」と鳴る現象を再現
  （WebAudioで合成、音源ファイル不要。号数で音量・音程、距離で減衰）。

## 内蔵データ（号数 → 打上高度 / 開花直径）
| 号数 | 打上高度 | 開花直径 |
|------|--------:|--------:|
| 3号 | 約125 m | 約65 m |
| 5号 | 約190 m | 約170 m |
| 10号(尺玉) | 約330 m | 約320 m |
| 15号(尺五寸玉) | 約410 m | 約440 m |
| 20号(二尺玉) | 約470 m | 約510 m |
| 30号(三尺玉) | 約600 m | 約620 m |

各大会は会場（緯度経度）と号数構成で特徴づけています。例: 隅田川=尺玉中心の大規模、
立川=都内最大の一尺五寸玉(15号)、ディズニー=園内の小型・低空花火（2〜4号）。
**打上場所の緯度経度は公開情報（河川敷・公園・球場等）からの概算**で、号数構成は各大会の
特徴に基づく代表値です（演出のため一部簡略化）。

## 技術
- [A-Frame](https://aframe.io/) 1.5.0（Three.js）＋ Three.js 粒子系
- 標準 Web API: getUserMedia（カメラAR）/ Geolocation / DeviceOrientation / WebAudio
- ホスティング: GitHub Pages（GitHub Actions, `.github/workflows/deploy.yml`）


## ファイル構成
```
index.html            リスト/マップ/AR画面＋カメラ映像＋A-Frameシーン
js/festival-data.js   9大会データ(座標・日付・号数構成)・号数スペック表
js/audio.js           開花音の合成(WebAudio)・距離遅延/減衰
js/geo.js             距離/方位(Haversine)・3D座標変換・GPS取得
js/fireworks.js       花火描画コンポーネント(hanabi-show): 上昇→実寸開花→重力/残光
js/gyro-look.js       ジャイロ見回し(gyro-look): 端末の傾きでカメラ姿勢（スワイプなし）
js/aspect-lock.js     アスペクト比ロック(aspect-lock): 横潰れ防止
js/north-align.js     真北整合(north-align): コンパスで北を一度だけ合わせる
js/guide.js           方向ガイド(direction-guide): ↓/←/→/後ろ のビルボード案内
js/app.js             制御: リスト(日付/距離順)→マップ(Leaflet)→許可→AR表示
（地図タイルは OpenStreetMap、Leaflet 1.9.4 を CDN 読込）
```

## デプロイ
対象ブランチ（`main`）への push で GitHub Actions が自動公開します。
ワークフローが Pages を自動有効化（`configure-pages` の `enablement: true`）するため、
**Settings の手動切替は不要** です。公開URLは Actions 実行ログ／Settings → Pages で確認できます。
