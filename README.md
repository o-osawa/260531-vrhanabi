# VR花火ビューア｜隅田川花火大会

スマートフォンのブラウザで、**隅田川花火大会の実データ**をもとに花火の3Dアニメーションを
VR（ジャイロ見回し）表示します。端末のGPSで現在地を取得し、打上場所までの **方位・距離** を
計算。コンパス（方位センサー）で **実際の方角** に花火を配置し、距離に応じた **見かけの大きさ**
で描画して「いま自分がいる場所からの見え方」を忠実に再現します。

## 使い方（スマホ）
1. GitHub Pages で公開されたHTTPSのURLをスマホのブラウザで開く
2. ［花火を見る］をタップ → 位置情報(GPS)と方位センサーの使用を許可
3. スマホを夜空にかざして傾けると、実際の方角・距離・サイズで花火が見えます
4. 方角がずれる場合は、主会場の方を向いて［方角を較正］をタップ

> センサー（GPS・方位）は **HTTPS** 上でのみ動作します。ローカルでは `http://localhost` も可。

## 仕組み（忠実再現）
- **実スケール座標系**（1 unit = 1 m）。視点をワールド原点に置き、各会場を
  方位・距離から `x = d·sinθ, z = -d·cosθ`（北=-Z）に配置、開花を実際の高度 `y = h` に置く。
  開花球の半径＝開花直径/2[m]。遠近投影により **見かけの画角（大きさ）が自動的に正しく** なる。
- **真北整合**: iOS `webkitCompassHeading` / Android `deviceorientationabsolute` から方位を取得し、
  花火ワールドを一定角回転させて北を実際の北へ合わせる（look-controls がジャイロ見回しを担当）。

出典: ホームメイト・リサーチ／なにわ淀川花火大会／全国花火データベース（号数仕様）、
中央区・ウォーカープラス（開催情報）。値は代表値で、演出のため一部丸めています。

## 技術
- [A-Frame](https://aframe.io/) 1.5.0（WebXR/Three.js）＋ Three.js 粒子系
- 標準 Web API: Geolocation / DeviceOrientation
- ホスティング: GitHub Pages（GitHub Actions, `.github/workflows/deploy.yml`）

## ローカル確認
```sh
python3 -m http.server 8080
# ブラウザで http://localhost:8080 を開く（PCはマウスドラッグで見回し）
```

## ファイル構成
```
index.html            A-Frameシーン＋起動UI/HUD
js/festival-data.js   隅田川データ・号数スペック表・打上プログラム
js/geo.js             距離/方位(Haversine)・3D座標変換・GPS取得
js/fireworks.js       花火描画コンポーネント(hanabi-show): 上昇→実寸開花→重力/残光
js/north-align.js     真北整合(north-align): コンパス/手動較正
js/app.js             制御: 許可要求→会場配置→開始→HUD更新
```

## デプロイ設定（初回のみ）
リポジトリの **Settings → Pages → Source** を **「GitHub Actions」** に設定すると、
対象ブランチへの push で自動公開されます。
